# Claw Deploy

基于 OpenClaw 的前端项目可视化打包部署桌面工具。

## 当前状态

当前仓库已完成第一阶段工程骨架：

- `Vue 3 + Vite + TypeScript` 前端基础结构
- 工作区布局、路由和 Pinia 状态管理
- `src-tauri` 最小目录骨架
- 基于 Tauri command 的项目路径识别能力
- 一套可直接用于桌面运行的临时应用图标资源
- `docs/` 中的 MVP 方案和开发拆解文档

## 本地运行前提

前端运行：

1. 安装依赖
2. 执行 `pnpm dev`

Tauri 运行额外要求：

1. 安装 Rust 工具链
2. 安装 Tauri 对应系统依赖
3. 执行 `pnpm tauri dev`

## 当前已实现能力

- 工作区基础布局
- 项目列表本地持久化
- 手动输入本地项目路径后识别 `package.json`
- 原生目录选择器导入本地项目
- 项目列表切换、最近使用排序和删除记录
- 项目默认打包配置与 dev/test/prod 环境配置表单
- OpenClaw WebSocket 网关连接面板与基础日志回显
- 网关连接配置由应用本地保存，适配公共用户使用
- 自动识别项目类型、包管理器、默认打包命令和默认输出目录

## 当前限制

- 项目导入依赖 Tauri 桌面环境，纯 `pnpm dev` 浏览器模式下无法读取本地目录
- 环境密码当前仍为本地明文存储，仅适合 MVP 阶段
- 网关 token 当前仍为应用本地明文存储，仅适合 MVP 阶段
- 任务执行协议、真实部署链路和 AI 诊断尚未开始接入
